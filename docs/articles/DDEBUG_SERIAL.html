Fangzheng Liu

I normally use VSCode with PlatformIO to develop Arduino. When developing feather M0 board with ATSADM21G18, remember to use "build_flags = -DDEBUG_SERIAL" in platform.ini. So the "Serial" will only be the USB port, and you can use the "#PIN_SERIAL_RX" and "PIN_SERIAL_TX" as noraml GPIO to control other things. Without using this, the "Serial.print" will output to both of the USB and the UART port. Which will mess up the other logic if you are using the UART pins as GPIO. "build_flags = -DUSE_TINYUSB" seems work, too.
